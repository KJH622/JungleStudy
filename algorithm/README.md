# Algorithm Study

알고리즘 문제를 풀고, 풀이 과정에서 배운 점과 실수한 부분을 기록하기 위한 저장소입니다.

정답 코드를 많이 쌓는 것보다,
왜 막혔는지와 다음에 무엇을 주의해야 하는지를 정리하는 데 집중합니다.

## Goal
- 알고리즘 문제 풀이 습관 만들기
- 자주 사용하는 자료구조와 알고리즘 익히기
- 틀린 문제와 헷갈린 문제를 복습할 수 있도록 기록 남기기

## Rule
- 문제를 먼저 혼자 고민한다.
- 바로 풀리지 않더라도 20~30분은 스스로 시도한다.
- 해설을 봤다면 반드시 다시 구현해본다.
- 기록은 길게 쓰지 않고 핵심만 남긴다.
- 맞은 문제라도 설명이 안 되면 다시 본다.

## How I Record
문제를 풀고 아래 내용만 간단히 기록합니다.

- 문제 분류
- 핵심 아이디어
- 내가 막힌 이유
- 구현 시 주의할 점
- 다시 풀 때 체크할 것

## Review Rule
- 해설을 본 문제는 다음 날 다시 풀기
- 헷갈렸던 문제는 1주 안에 다시 보기
- 구현 실수가 있었던 문제는 따로 체크하기

## Study Flow
1. 문제 읽기
2. 사용할 자료구조나 알고리즘 생각하기
3. 직접 구현하기
4. 막힌 이유와 핵심 포인트 기록하기
5. 다시 풀기

## Memo
이 저장소의 목적은 정답을 모으는 것이 아니라,
문제를 풀면서 부족했던 점을 확인하고 반복해서 보완하는 것입니다.
